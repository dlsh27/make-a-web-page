Lesson 1: The Basics of the Web and HTML

The Worldwide Web

The Worldwide Web is primarily a collection of HTML documents.

Major Components of the WWW

  • Your computer and browser (MS IE, Chrome, Firefox, etc.).
  • The Internet (world's largest computer network).
  • Server (computer that receives request using HTTP protocol and sends information back to browser).

HTML Documents

HTML documents are the heart of the Web.

HTML documents are made up of elements. Browsers recognize these elements through the use of tags.

Tags

Tags can be used to format text, add images and or create links to web pages.
Here are examples: Bold, Italics or Sports.

DIV and SPAN

These are known as block and inline elements, respectively. Block elements, the DIV tag, create an invisible box around content. Inline elements, the SPAN tag, creates a box within a line of text.

Oh, Those Stupid Computers

Computers are stupid! What is meant by this is that computers interpret instructions literally. This can turn small programming errors into large problems.

Lesson 2: Creating a Structured Document with HTML

Coding Languages Used for Web Pages

  • HTML (Hyper Text Manipulation Language)- Used to lay out and structure web page (like walls in a house)
  • CSS (Cascading Style Sheets)- Used to define the style of web pages (like carpet, paint & wall decorations in a house)
  • JavaScript - Controls interactive components on a page (similar to a garage door opener or TV remote in a home)

Web Page Structures

  • Developer tools in browsers can be used to view their underlying structure.
  • Elements, tags, objects, images, etc. appear in a tree-like structure known as Document Object Model (DOM).
  • All elements on web pages are contained in boxes. The boxes can contain text, images and/or shapes.

Code & Indentations

Code is written using a series of indentations. This is done to make it more readable and, therefore, easier to debug. Indentations are also used to depict the nested nature boxes in HTML.

Text Editors

Text editors are tools that are used as a venue for writing code. The three tools evaluated were Scratchpad, Codepen and Sublime Text. Each tool has relative strengths & weaknesses, so choosing one is really a matter of personal preference.

Lesson 3: Adding Style Elements to HTML Structure

CSS

CSS stands for Cascading Style Sheets. The primary function of CSS is to add style elements to HTML.

Code, Test, Refine

This is the methodology that we have been following so far as we write code. The steps are:

1) Look for natural boxes.
2) Look for repeated styles & semantic elements.
3) Write HTML.
4) Apply styles (biggest features on page to smallest features).
5) Fix things.

Dev Tools are used to examine, evaluate and correct code on the fly.

Avoiding Repetition

Avoiding repetition is important for several reasons:
  • Allows consistancy when applying a style or change across a group of elements.
  • Eliminates the need to examine many, stand-alone lines of code to find mistakes.
  • Less code writing is required, generating more usable code.
  • Saves work for the developer.
